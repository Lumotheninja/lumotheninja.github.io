Problem statement

There are a number of challenges associated with the PascalVOC 2012 dataset, which is obtained from Flickr. The tasks include object classification (multilabel classification), object detection, object segmentation, human layout and action classification. The main goal of this challenge is to recognize objects from a number of visual object classes in realistic scenes (i.e. not pre-segmented objects). It is fundamentally a supervised learning learning problem in that a training set of labelled images is provided.

To tackle the object classification problem , we train a neural network to automatically predict the label(s) for an image. This is a multilabel classification problem as there might be multiple classes for an image.

Exploratory Data Analysis

There are 11,530 images in the PascalVOC 2012 dataset consisting of the following 20 classes: person, bird, cat, cow, dog, horse, sheep, aeroplane, bicycle, boat, bus, car, motorbike, train, bottle, chair, dining table, potted plant, sofa and monitor/tv. Besides the images, a file containing the annotations in XML format is also provided.

A check with a simple script also reveals all images to be 3-channel coloured. With the data being very clean, and the features being only the images, we can then apply the standard deep learning methods.

Implementation

Transfer learning

We used the Resnet34 neural network architecture loaded using Pytorch’s model zoo. The Resnet has been pretrained with the Imagenet dataset, so we used transfer learning by keeping the pretrained weights and changing only the final fully connected layer.

model = models.resnet34(pretrained=True)
model.fc = torch.nn.Linear(512, 20)

Data augmentation

During training, we processed the data fed into our neural network by the following transformations. The crops are necessary to make the it a standardized input and the five crop is to let the classifier see the edges of the images. The final transform is normalizing the data to the mean and standard deviation of the Imagenet dataset since we are using transfer learning.

transforms.FiveCrop(224)
transforms.ToTensor()
transforms.Normalize(mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])

Example of objects at the corner

During evaluation, we did not use five crop. The reason being that if the 5 evaluations yield different values and if we add them up, we might end up adding too much noise to our output. Thus, we decided to only use center crop with the risk that we might miss the objects at the corners.

transforms.Resize(size=280)
transforms.CenterCrop(224)
transforms.ToTensor()
transforms.Normalize(mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])

The results

We achieved pretty good results for our classifier, achieving mean average precision of 0.844 and number 20 on the global leaderboard. Part of our success comes from the fact that we have done many data augmentation such as five crop for training and transfer learning from the Imagenet dataset which leads to more robust classification. Although this is a pretty old challenge (it ended in 2012), I learnt a lot about deep learning, particularly on the power of transfer learning and feature transformation and how you don’t have to use the latest models to achieve good results.

Learnings

  1. Tech used: pytorch and flask.
  2. Data augmentation is very important to an image-related task, maybe even more than finetuning the model
  3. A requirements of the projects is to have a GUI to view the images classified, such visualizations have made us realize what sorts of augmentations are suitable such as five crop.
