Java Pass By Reference

20 Nov 2019 on Java

How a Java snippet about serialization taught me about object references

I have always heard people mentioning about Java, and how Java Objects pass by reference, while Java primitives pass by value. I think it can be simply compared to mutable and immutable objects from Python. Consider the code snippet below:

//Deserialization bomb - deserializing this stream takes forever
static byte[] bomb() {
    Set<Object> root = new HashSet<>()
    Set<Object> s1 = root;
    Set<Object> s2 = new HashSet<>();
    for (int i = 0; i < 100; i++) {
        Set<Object> t1 = new HashSet<>();
        Set<Object> t2 = new HashSet<>();
        t1.add("foo"); // Make t1 unequal to t2
        s1.add(t1); s1.add(t2);
        s2.add(t1); s2.add(t2);
        s1 = t1;
        s2 = t2;
    }
    return serialize(root); // Method omitted for brevity
}

Source: Bloch, Effective Java, pg. 340

While Bloch is trying to teach us about the dangers of deserialization, what I saw was instead the way the variable root was changing. If one were to insert a print statement into the for loop, you would get the following printouts:

[[], [foo]]
[[[], [foo]], [[], foo, [foo]]]
[[[[], [foo]], [[], foo, [foo]]], [[[], [foo]], foo, [[], foo, [foo]]]]
[[[[[], [foo]], [[], foo, [foo]]], [[[], [foo]], foo, [[], foo, [foo]]]], [[[[], [foo]], [[], foo, [foo]]], foo, [[[], [foo]], foo, [[], foo, [foo]]]]]

But hey, how is root getting modified? didn’t we just add t1 and t2 to s1, and then we even set s1 to t1. Shouldn’t the reference to root be totally lost???

One of the revealing clues is that although root is changing, root always has 2 children. This means that the elements are added to root the first time, but it has changed. This tricky code is a combination of a cool recursive-like structure as well as the Java pass by reference. If we were to convert the code to Python and treat HashSet as a list, we can see the corresponding code as:

root = []
s1 = root
s2 = []
for i in range(100):
    t1 = []
    t2 = []
    t2.append("foo")
    s1.append(t1)
    s1.append(t2)
    s2.append(t1)
    s2.append(t2)
    s1 = t1
    s2 = t2

Seeing this code makes it clear. In the first iteration, root/s1 is a list of [] and [“foo”] which is t1 and t2 respectively, however, when you set s1=t1, the new s1 is the element inside root, the [], and when you append to it, you essentially modify it. Running the code will yield the same result as Java. This problem is tricky, but is not hard to understand once you write in out step by step and understand how Java passes by reference. After all, typesafe just means that you can’t easily convert an Object to another class, it does not mean that something is always invariant and immutable.

TIL:

  1. Java pass by reference works for Objects which is kinda like having a mutable object in Python. Similarly, Java pass by value is kinda like immutable objects
  2. Typesafe for Java only means safety for class conversion and not mutable objects.
